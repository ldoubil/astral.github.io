## 📌 前提条件

- **有一台电脑** 
- **有一台带有ipv4公网的服务器** 

## 📦 ET服务端部署教程

> 📋 **示例环境：** Ubuntu 22.04 LTS（其他Linux操作系统类似）

 ### 🔑 使用ssh工具连接到服务器

> 📋 **示例环境：** ssh工具为：[WindTerm](https://github.com/kingToolbox/WindTerm/releases)（其他ssh工具类似）（下载可能需要科学上网）

> ⚠️ **重要提醒：**  
> 1. 您打开的WindTerm界面可能与图示略有不同
> 2. 所有服务器信息请以服务器提供商提供的信息为准

**Step 1：** 进入WindTerm，点击左上角**会话**

[进入ssh工具](/images/服务器部署/服务器部署-1.png)

**Step 2：** 在弹出的窗口中，**主机**部分填入服务器提供商给你的ip地址，**端口**处填写服务器提供商给你的ssh端口，随后点击下方**连接**

> ⚠️ **重要提醒：** 一般ssh端口默认为**22**,但服务器提供商可能会为了安全进行更改

[ssh连接](/images/服务器部署/服务器部署-2.png)

**Step 3：** 在弹出的**登录**窗口中，填写服务器提供商给你的用户名，随后点击下方的**继续**

> ⚠️ **重要提醒：** 一般用户名默认为**root**,但服务器提供商可能会为了安全进行更改

💡 **可选设置：** 勾选下方的**下次自动登录**

[ssh连接](/images/服务器部署/服务器部署-3.png)

**Step 4：** 随后输入服务器提供商给你的ssh密码，随后继续点击右下方的**继续**

[ssh连接](/images/服务器部署/服务器部署-4.png)

**最终连接效果** 

> ⚠️ **重要提醒：** 可能会有点不一样，因为所使用的系统不同

[ssh连接](/images/服务器部署/服务器部署-5.png)

> ✅ **服务器连接完成！** 恭喜你成功连接到你的服务器了，现在可以进行下一步了

### ⚙️ 安装ET服务端

**Step 1：** 使用该指令更新apt

> ⚠️ **重要提醒：** 如果该命令无法更新请自行查找更新指令

```
sudo apt update 
```

**Step 2：** 检查是否已安装unzip，在WindTrem中的服务器窗口中输入

```
unzip -v 
```
 
[安装unzip](/images/服务器部署/服务器部署-6.png)

**Step 3：** 根据提示安装unzip

> ⚠️ **重要提醒：** 如果该命令无法安装请自行查找安装指令

```
apt install unzip
```

#### 关于乌班图系统，如果不是请跳到**Step 4：**

**如果你使用的是乌班图（Ubuntu）系统，那么安装unzip时将会弹出此窗口，请不要慌张，点击键盘左上角的**ESC**键退出即可**

[重启内核](/images/服务器部署/服务器部署-7.png)

**最后来到这，请继续下一步操作**

[重启内核](/images/服务器部署/服务器部署-8.png)

**Step 4：** 检查unzip是否安装成功，如下图所示

```
unzip -v 
```

[安装unzip](/images/服务器部署/服务器部署-9.png)

**Step 5：** 安装EasyTier

**一键安装命令**

```
wget -O /tmp/easytier.sh "https://raw.githubusercontent.com/EasyTier/EasyTier/main/script/install.sh" && sudo bash /tmp/easytier.sh install --gh-proxy https://ghfast.top/
```

**最终安装效果**

[安装ET](/images/服务器部署/服务器部署-10.png)

**Step 6：** 启动EasyTier

```
systemctl start easytier@default
```

### 修改EasyTier配置文件

**Step 1：** 将以下配置复制到配置文件中

```
hostname = "" #更改想要的服务器名称,留空将随机生成
dhcp = true
listeners = [
    "tcp://0.0.0.0:11010",
    "udp://0.0.0.0:11010",
]
rpc_portal = "127.0.0.1:15888"

[network_identity]
network_name = ""
network_secret = ""

[flags]
enable_kcp_proxy = true
enable_quic_proxy = true
latency_first = true
```

**Step 2：** 随后会弹出一个窗口，点击**是**即可

> ⚠️ **重要提醒：** 如果你使用的是别的ssh工具，可能需要下载的自己电脑上更改后再上传回服务器替换

[保存](/images/服务器部署/服务器部署-15.png)

**Step 3：** 输入该指令重启EasyTier服务

```
systemctl restart easytier@default
```

[重启](/images/服务器部署/服务器部署-18.png)

> ✅ **完成EasyTier的文件配置！** 恭喜您已完成EasyTier的文件配置了，现在可以进行下一步了

### 📱 使用Astral连接到您的服务器

> ⚠️ **重要提醒：** Astral简称AS，后续教程中会简称为AS

**Step 1：** 打开AS，点击左侧的**服务器**，随后点击右下角的**加号**

[添加服务器](/images/服务器部署/服务器部署-16.png)

**Step2 ：** 在弹出的窗口中，**服务器名称**用于在**服务器**页面中区分您的服务器，**服务器地址**填写您服务器的公网IP地址加上**:11010**，随后点击右下方的**添加**

> ⚠️ **重要提醒：** 服务器地址中的端口号**默认为11010**，如您已更改请改为您所更改的

> ⚠️ **重要提醒：** 端口号前为**英文冒号**，请勿使用中文冒号

[添加服务器](/images/服务器部署/服务器部署-17.png)

**Step3 ：** 随后您便可以像公益服务器一样正常使用您的服务器了，如果您需要更安全的服务器配置，请继续看下方的进阶教程

> ⚠️ **重要提醒：** 如果您不希望把服务器给陌生人用，那么请不要随意分享您的服务器地址，并告知您的朋友不要随意分享

## 🛡️ 进阶教程

### 将服务器变成私有模式

> ⚠️ **重要提醒：** 进阶教程主要目的是将服务器变成私有模式，避免陌生人白嫖

**Step 1：** 在EasyTier配置文件中的**底部额外添该行**

```
private_mode = true
```

**Step 2：** 并修改**[network_identity]**部分

> ⚠️ **重要提醒：** 如果您使用的是WindTerm，那么您的每次修改都会触发WindTerm的自动上传机制，所以建议您在自己电脑上输入好再复制进去

> ⚠️ **重要提醒：** 如果你使用的是别的ssh工具，可能需要下载的自己电脑上更改后再上传回服务器替换

**
network_name = "" #输入您打算在AS使用的房间名称

network_secret = "" #输入您打算在AS使用的房间密码
**

**Step 3：** 在弹出的窗口中，点击**是**即可

[保存](/images/服务器部署/服务器部署-15.png)

**Step 4：** 在WindTerm中输入该指令重启EasyTier服务

```
systemctl restart easytier@default
```

[重启](/images/服务器部署/服务器部署-18.png)

> ✅ **完成EasyTier文件的私人配置！** 恭喜您已完成EasyTier的私人配置了，现在可以进行下一步了

### 在AS中创建私人房间

**Step 1：** 打开AS，点击左侧的**房间**，随后点击右下角的**加号**，**房间名称**用于**区分**房间**页面的其他房间**，**关闭保护模式**，在**房间号**中填写你刚刚在**[配置文件](#将服务器变成私有模式)中**设置的名称，**房间密码**同样,最后点击确定

[添加房间](/images/服务器部署/服务器部署-19.png)

**Step 2：** 最后返回**主页**点击右下角的**连接**即可

> ⚠️ **重要提醒：** 如果无法连接到服务器，请检查房间名称和密码是否相同，如果已确认相同请尝试**重启AS**重新连接

> ⚠️ **重要提醒：** 当您点击AS右上角的**叉号**时，AS并不会退出，您需要点击**右下角箭头**展开后台软件，在弹出的窗口右键AS图标，并点击**退出**

[退出](/images/服务器部署/服务器部署-20.png)

> ✅ **完成连接！** 恭喜您已成功连接上房间了
