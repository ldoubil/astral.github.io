服务器是 Astral 网络的重要基础设施，它们帮助设备之间建立连接并提供中继服务。

## 📡 什么是 Astral 服务器？

Astral 服务器充当网络中继和协调中心：

- 🔄 **连接协调**：帮助设备发现彼此
- 🌉 **网络中继**：在直连失败时提供数据转发
- 📍 **地理分布**：提供就近访问，降低延迟
- 🆓 **免费使用**：社区提供的公共服务器

## ⚙️ 服务器配置步骤

<Steps>
1. **进入服务器管理**
   - 点击左侧菜单的**"服务器"**选项

2. **选择服务器**
   - 点击右下角的**地球图标** 🌍
   - 浏览可用服务器列表

3. **启用服务器**
   - 选择上方的三个服务器
   - 点击**"添加"**按钮

4. **启用和配置协议**
   - 在房间页面点击服务器名称右边的开关，启用服务器随后点击右边的**"编辑"**按钮
   - 确保同时勾选 **TCP** 和 **UDP** 协议
   - 保存设置
</Steps>

## 🎯 如何选择最佳服务器？

### 1. 延迟测试
```bash
# Windows 用户可以使用 ping 命令测试
ping turn.sc.629957.xyz

# 或者使用 Astral 内置的延迟显示
```

### 2. 地理位置考虑
- 🇨🇳 **中国大陆用户**：推荐选择国内服务器
- 🇭🇰 **港澳台用户**：可选择香港服务器
- 🌍 **海外用户**：选择官方的服务器

### 3. 服务器特性
- **IPv4/IPv6**：部分服务器支持双栈
- **负载均衡**：选择负载较低的服务器
- **稳定性**：优先选择运行时间长的服务器

## 🔧 高级服务器设置

### 协议配置
- **TCP**：可靠传输，适合文件传输
- **UDP**：低延迟，适合游戏和实时应用
- **建议**：同时启用两个协议

### 连接优化
- **延迟优先**：在设置中开启此选项
- **自动重连**：网络中断时自动重新连接

## 📊 服务器状态监控

在 Astral 中您可以查看：

- 📡 **连接延迟**：到服务器的网络延迟
- 📈 **连接质量**：网络连接稳定性
- 👥 **在线用户**：当前连接的用户数量
- 🔄 **服务器负载**：服务器资源使用情况

## 🚨 服务器问题排查

### 连接失败
1. **检查网络连接**
   - 确保本地网络正常
   - 测试其他网站的连通性

2. **尝试其他服务器**
   - 切换到不同地区的服务器
   - 对比连接效果

3. **检查防火墙**
   - 确保防火墙已关闭
   - 或添加 Astral 到白名单

### 连接不稳定
1. **网络环境检查**
   - 检查本地网络稳定性
   - 检查在网络低峰期使用是否正常

2. **服务器切换**
   - 尝试负载较低的服务器
   - 选择距离更近的服务器

## 🔄 服务器切换策略

### 日常使用
- **固定选择**：选择一个稳定的服务器长期使用
- **备选方案**：选择2-3个备用服务器

### 多人协作
- **统一选择**：所有团队成员使用同一服务器
- **提前协调**：在开始前确认服务器是否启用

### 性能优化
- **定期测试**：定期测试不同服务器的性能
- **灵活调整**：根据网络状况灵活切换

- 游戏联机时，确保所有玩家选择相同服务器
