# 🏠 房间管理

房间是 Astral 的核心功能，掌握房间管理技巧能让您更好地使用网络功能。

## 🎯 房间概念深入理解

  <Card title="🌐 虚拟局域网" icon="puzzle">

  </Card>

  </Card>

## 🛠️ 高级房间配置

### 房间类型选择

**普通房间**
- 🔓 房间信息可见
- 📱 可通过房间号直接加入
- 💡 适合公开的协作环境

**保护房间**
- 🔒 房间信息加密隐藏
- 📊 只能通过分享码加入
- 🛡️ 提供更高的安全性

### 详细配置参数

<Steps>
1. **基本信息设置**
   - **房间名称**：用于区分别的房间
   - **房间号**：可以是数字、字母或组合

2. **安全设置**
   - **访问密码**：设置复杂密码提高安全性
   - **保护模式**：启用加密保护

3. **网络配置**
   - **IP地址段**：自定义虚拟网络地址范围
</Steps>

## 👥 成员管理

### 查看成员状态
在房间页面可以查看：
- 📱 **设备名称**：成员的设备名称
- 🌐 **IP地址**：在虚拟网络中的地址
- 📊 **连接状态**：在线/离线状态
- 📡 **网络质量**：延迟和丢包状况

### 成员权限管理
- 👑 **房主权限**：完全控制房间设置
- 👤 **普通成员**：只能连接和使用网络

## 🔧 房间网络配置

### IP地址分配
```
默认网段：10.126.126.0/24
- 房主通常获得：10.126.126.1
- 成员依次分配：10.126.126.2、10.126.126.3...
- 广播地址：10.126.126.255**（256和257以此往后推的所有地址不可用）**
```

### 端口管理
- **游戏端口**：25565（Minecraft）、27015（Steam游戏）
- **文件共享**：445（SMB）、21（FTP）
- **远程桌面**：3389（RDP）、5900（VNC）
- **自定义端口**：根据应用需求配置

## 📊 房间监控与诊断

### 连接质量监控
- **延迟监控**：实时查看网络延迟
- **丢包率**：监控数据传输质量
- **带宽使用**：查看网络流量情况
- **连接稳定性**：评估网络连接可靠性

### 问题诊断工具
```bash
# 测试连通性
ping 10.126.126.1

# 检查路由
tracert 10.126.126.1

# 端口测试
telnet 10.126.126.1 25565
```

## 🔄 房间生命周期管理

### 房间创建最佳实践
1. **规划阶段**
   - 确定房间用途和参与人数
   - 选择合适的服务器
   - 制定命名规范

2. **配置阶段**
   - 设置有意义的房间名称
   - 配置适当的防火墙策略
   - 测试网络连通性

3. **使用阶段**
   - 定期检查成员状态
   - 监控网络性能
   - 及时处理连接问题

### 房间维护
- 🧹 **定期清理**：删除不再使用的房间
- 🔄 **配置更新**：根据需求更新
- 👥 **成员管理**：定期审查成员列表
- 📊 **性能优化**：启用延迟优先

## 🛡️ 安全最佳实践

### 密码策略
- 📏 **长度要求**：至少8位字符
- 🔀 **复杂性**：包含数字、字母和符号
- 🔄 **定期更换**：定期更新房间密码
- 🚫 **避免泄露**：不在公开场所讨论密码

## 🚨 常见问题处理

### 房间连接问题
1. **无法加入房间**
   - 检查房间号和密码
   - 确认服务器选择一致
   - 验证网络连接状态

2. **成员看不见彼此**
   - 检查防火墙设置
   - 尝试重新连接
   - 更换服务器测试
   - 尝试重启软件

3. **网络延迟过高**
   - 选择更近的服务器
   - 检查本地网络质量
   - 启用延迟优先选项

---

<Aside type="tip" title="房间管理技巧">
- 为不同用途创建不同的房间
- 使用有意义的命名方式便于管理
- 定期清理不活跃的房间
- 保存重要房间的房间号和密码信息
</Aside>
