## 🏢 企业内网访问

### 场景描述
员工在家需要访问公司内部的：
- 📊 **业务系统**：ERP、CRM、财务系统等
- 📁 **文件服务器**：共享文档、项目资料
- 🖨️ **网络设备**：打印机、网络存储等
- 💻 **开发环境**：内网开发服务器、数据库

### 部署方案

<Tabs>
  <TabItem label="简单部署">
    **适用场景**：小型团队（5-10人）

    <Steps>
    1. **在公司设置网关设备**
       - 在公司网络中部署一台运行 Astral 的设备
       - 配置为企业房间的"网关节点"

    2. **员工设备配置**
       - 在个人设备上安装 Astral
       - 加入企业房间
       - 配置路由表指向公司内网

    3. **网络访问测试**
       - 测试访问公司内网 IP 地址
       - 验证各种业务系统的连通性
    </Steps>
  </TabItem>

  <TabItem label="高级部署">
    **适用场景**：中大型企业（10人以上）

    <Steps>
    1. **搭建专用服务器**
       - 部署企业专用的 Astral 服务器
       - 配置企业级安全策略

    2. **分层网络架构**
       - 设置不同部门的独立房间
       - 配置跨房间访问权限

    3. **安全策略实施**
       - 启用强制加密传输
       - 配置访问控制列表
       - 实施多因素认证

    4. **监控和管理**
       - 部署网络监控系统
       - 配置日志审计功能
    </Steps>
  </TabItem>
</Tabs>

### 配置示例

**路由配置**：
```bash
# 添加到公司内网的路由
route add 192.168.1.0 mask 255.255.255.0 10.126.126.1

# 添加到特定服务器的路由
route add 172.16.0.100 mask 255.255.255.255 10.126.126.1
```

**DNS 配置**：
```
# 公司内部 DNS 服务器
Primary DNS: 192.168.1.10
Secondary DNS: 192.168.1.11
```

## 📁 团队文件协作

### 共享文件夹设置

<Tabs>
  <TabItem label="Windows 文件共享">
    **配置 SMB 文件共享**：

    <Steps>
    1. **设置共享文件夹**
       - 右键点击要共享的文件夹
       - 选择"属性" → "共享"选项卡
       - 点击"高级共享"

    2. **配置共享权限**
       - 勾选"共享此文件夹"
       - 设置共享名称
       - 点击"权限"配置访问权限

    3. **设置安全权限**
       - 切换到"安全"选项卡
       - 添加要授权的用户或组
       - 设置适当的权限级别
    </Steps>

    **访问共享文件夹**：
    ```
    # 在文件资源管理器中输入
    \\10.126.126.1\ShareFolder

    # 或使用命令行映射网络驱动器
    net use Z: \\10.126.126.1\ShareFolder
    ```
  </TabItem>

  <TabItem label="跨平台解决方案">
    **使用 FTP 服务器**：

    <Steps>
    1. **安装 FTP 服务器**
       - Windows：IIS FTP 服务
       - Linux：vsftpd 或 ProFTPD
       - 第三方：FileZilla Server

    2. **配置 FTP 服务**
       - 设置用户账户和密码
       - 配置访问目录
       - 启用安全传输（FTPS/SFTP）

    3. **客户端连接**
       - 使用 FTP 客户端连接
       - 地址：ftp://10.126.126.1
       - 输入用户名和密码
    </Steps>
  </TabItem>
</Tabs>

### 版本控制协作

**Git 仓库共享**：
```bash
# 在服务器端创建裸仓库
git init --bare project.git

# 团队成员克隆仓库
git clone http://10.126.126.1/project.git

# 配置远程仓库
git remote add origin http://10.126.126.1/project.git
```

**SVN 服务器搭建**：
```bash
# 创建 SVN 仓库
svnadmin create /var/svn/repos/project

# 启动 SVN 服务
svnserve -d -r /var/svn/repos

# 客户端连接
svn checkout svn://10.126.126.1/project
```

## 🖥️ 远程桌面控制

### Windows 远程桌面

<Steps>
1. **启用远程桌面**
   - 右键"此电脑" → "属性"
   - 点击"高级系统设置"
   - 在"远程"选项卡中启用远程桌面

2. **配置防火墙**
   - 确保远程桌面端口（3389）开放
   - 或临时关闭防火墙

3. **连接远程桌面**
   - 使用"远程桌面连接"工具
   - 输入目标 IP：10.126.126.1
   - 输入用户名和密码
</Steps>

### 企业级安全配置

<Steps>
1. **双因素认证**
   - 配置 TOTP 动态密码
   - 集成企业 AD/LDAP 认证
   - 设置登录失败限制

2. **网络隔离**
   - 分离生产和测试环境
   - 设置 DMZ 区域
   - 实施最小权限原则

3. **数据保护**
   - 启用磁盘加密
   - 配置自动备份
   - 实施数据丢失防护

4. **合规性要求**
   - 遵守行业安全标准
   - 实施数据保留政策
   - 配置合规性报告
</Steps>

## 🚨 故障排除

### 常见问题处理

<Tabs>
  <TabItem label="连接问题">
    **症状**：无法连接到企业网络

    **排查步骤**：
    1. 检查 Astral 连接状态
    2. 验证房间配置正确性
    3. 测试网络连通性
    4. 检查防火墙设置
    5. 确认服务器可用性
  </TabItem>

  <TabItem label="性能问题">
    **症状**：访问速度慢，延迟高

    **优化方案**：
    1. 更换延迟更低的服务器
    2. 启用网络加速选项
    3. 优化本地网络环境
    4. 调整应用程序设置
    5. 升级网络带宽
  </TabItem>

  <TabItem label="安全问题">
    **症状**：怀疑存在安全风险

    **应对措施**：
    1. 立即更改所有密码
    2. 检查访问日志
    3. 扫描恶意软件
    4. 更新安全策略
    5. 联系安全专家
  </TabItem>
</Tabs>
