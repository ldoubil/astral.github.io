# 🚪 加入房间

如果您的朋友已经创建了房间，您可以通过房间号或分享码加入他们的虚拟网络。

## 🎯 加入前的准备

在加入房间之前，请确保：

- ✅ 已完成 [基础设置](/quick-start/basic-settings)
- ✅ 防火墙已关闭
- ✅ 用户名已设置且唯一
- ✅ 选择了与房主**相同的服务器**

<Aside type="caution" title="服务器选择重要提醒">
**必须选择与房主相同的服务器！** 这是成功加入房间的关键前提条件。
</Aside>

## 📱 两种加入方式

<Tabs>
  <TabItem label="房间号加入">
    ### 使用房间号和密码加入

    这是最常见的加入方式，需要房主提供房间号和密码。

    <Steps>
    1. **进入房间管理**
       - 点击左侧菜单的**"房间"**选项

    2. **添加新房间**
       - 点击**"加号"**图标 ➕

    3. **输入房间信息**
       - **房间名称**：给这个房间起个便于记忆的名字
       - **房间号**：输入房主提供的房间号
       - **房间密码**：输入密码（如果房间设置了密码）

    4. **确认加入**
       - 点击**"确定"**添加房间
    </Steps>

    ![加入房间界面](/images/食用-5.png)
  </TabItem>

  <TabItem label="分享码加入">
    ### 使用分享码快速加入

    如果房主启用了房间保护，您需要使用分享码加入。

    <Steps>
    1. **进入房间管理**
       - 点击左侧菜单的**"房间"**选项

    2. **使用分享码**
       - 点击右下角的**"任务板"**图标 📋

    3. **输入分享码**
       - 将房主发送的分享码粘贴到输入框
       - 点击**"确定"**

    4. **自动配置**
       - 系统会自动配置房间信息
       - 房间将出现在您的房间列表中
    </Steps>

    ![分享码加入](/images/MC联机/MC联机-24.png)
  </TabItem>
</Tabs>

## 🔗 连接到房间

添加房间后，您需要连接到它：

<Steps>
1. **选择房间**
   - 在房间列表中找到刚加入的房间
   - 点击房间使其呈现**选中状态**

2. **建立连接**
   - 返回主界面
   - 点击**"连接"**按钮

3. **等待连接**
   - 系统会尝试建立连接
   - 等待状态变为**"已连接"**
</Steps>

![连接状态](/images/食用-6.png)

## 📊 验证连接状态

连接成功后，检查房间页面：

### 正常连接状态
- 🖥️ **本机**：显示您的设备信息和IP地址
- 👥 **其他成员**：可以看到房主和其他成员
- 🌐 **服务器**：显示中继服务器信息
- 📡 **连接质量**：显示网络延迟和状态

![连接状态示例](/images/MC联机/MC联机-26.png)

### 连接问题排查
如果无法看到其他成员或连接失败：

1. **检查服务器选择**
   - 确保与房主选择了相同的服务器
   - 对比服务器地址是否一致

2. **检查网络设置**
   - 确认防火墙已关闭
   - 检查网络连接是否正常

3. **检查房间信息**
   - 验证房间号是否正确
   - 确认密码是否正确

## 🎮 开始使用网络

连接成功后，您就可以：

### 获取虚拟IP地址
在房间页面，用户名下方显示的IP地址就是您在虚拟网络中的地址：
- 📍 **示例**：`10.126.126.2`
- 🎯 **用途**：其他设备可以通过这个IP访问您的设备
- 🎮 **游戏联机**：在游戏中输入房主的IP地址进行连接

### 测试网络连通性
您可以使用 ping 命令测试与其他成员的连接：
```bash
ping 10.126.126.1
```

<Aside type="tip" title="关于IP地址">
虚拟网络中的IP地址通常以 `10.126.126.x` 的格式出现，每个设备会分配到不同的最后一位数字。
</Aside>

## 🔄 常见连接问题

### 问题1：看不到其他成员
**可能原因**：
- 服务器选择不一致
- 房间信息错误
- 网络连接问题

**解决方案**：
- 重新确认服务器地址
- 验证房间号和密码
- 重启应用后重新连接

### 问题2：连接速度慢
**可能原因**：
- 服务器延迟高
- 网络环境复杂
- NAT类型限制

**解决方案**：
- 尝试更换服务器
- 开启"延迟优先"设置
- 查看 [NAT类型说明](/troubleshooting/nat-types)

### 问题3：频繁断线
**可能原因**：
- 网络不稳定
- 防火墙干扰
- 服务器负载高

**解决方案**：
- 检查网络连接稳定性
- 确认防火墙完全关闭
- 更换其他服务器
